# AI Trading Radar v3.0.0

## Safe multi-pair configuration

Start from `config_v3.example.json`. The legacy `symbol` field still works;
adding `symbols` enables one isolated processor per enabled entry. Each entry
can override `scalping` values without affecting any other symbol.

`agent_execution_enabled` defaults to `false`. Agents still publish weighted
reasoning to the dashboard, but the established strategy and risk gates remain
the only entry path until this setting is deliberately enabled and paper-tested.

`gemini_enabled` defaults to `false`; an API key is required and the agent is
informational only. Never put a Gemini key in a committed config file.

ATR protection is also opt-in (`atr_risk_enabled`). When enabled, it bounds
entry SL/TP from current ATR; broker-side protection stays mandatory.

## Periodic optimizer

Run a bounded, offline grid search against a CSV dataset:

```powershell
python optimize_strategy.py --config config_v3.example.json --data data\xauusd.csv --output optimizer_report.json
```

Review the JSON report before copying a candidate to a paper config. The script
does not write to, reload, or apply any live configuration.

## Docker

```powershell
Copy-Item config_v3.example.json config.json
docker compose up --build
```

The provided container is for paper, CCXT, or Alpaca workflows. Native MT5
execution requires Windows and should remain outside this Linux container.
